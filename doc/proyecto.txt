PROYECTO 2
SIMULADOR DE EXPEDICIÓN ESPACIAL
Operación: Nova Horizon

Introducción
La humanidad ha logrado establecer una base científica en el planeta Nova Horizon, ubicado a cientos de años luz de la Tierra.
Sin embargo, la colonia depende de expediciones periódicas para obtener recursos, investigar nuevas zonas y garantizar la supervivencia de sus habitantes.
El objetivo de este proyecto es desarrollar un sistema que permita administrar una expedición espacial completa.
El usuario será el comandante de la misión y deberá tomar decisiones estratégicas para conformar el equipo de astronautas, administrar los recursos disponibles y seleccionar las misiones más adecuadas.
Todo el sistema deberá desarrollarse utilizando únicamente:
HTML5
Tailwind CSS 4.3
CSS (si es necesario)
JavaScript Vanilla

No se permite utilizar bases de datos ni tecnologías externas.

Contexto
La colonia Nova Horizon cuenta con recursos limitados.
Cada misión implica riesgos y consume recursos.
Los astronautas poseen habilidades diferentes.
Elegir un equipo inadecuado puede provocar el fracaso de la expedición.
El comandante deberá administrar correctamente todos los recursos disponibles para mantener la colonia funcionando.

Historia
La Agencia Espacial Internacional ha detectado señales provenientes de una zona inexplorada.
Se sospecha que allí existen enormes reservas de minerales capaces de solucionar la crisis energética de la Tierra.
Sin embargo, la región presenta condiciones extremadamente peligrosas.
La agencia necesita un comandante capaz de organizar expediciones exitosas.
Cada decisión afectará el desarrollo de la misión.

Objetivo General
Desarrollar un simulador de administración de expediciones espaciales donde el usuario gestione astronautas, recursos y misiones utilizando JavaScript Vanilla.

Reglas de Negocio
RN01
Un astronauta solo puede participar en una misión a la vez.

RN02
No puede iniciar una misión sin suficientes recursos.

RN03
Una misión consume combustible.

RN04
Una misión consume oxígeno.

RN05
Después de cada misión
La energía disminuye.

RN06
Si la energía llega a cero
El astronauta queda
"No Disponible".

RN07
Cada misión exitosa aumenta la experiencia.

RN08
La experiencia máxima es 100.

RN09
No se pueden repetir nombres de astronautas.

RN10
Los recursos nunca pueden ser negativos.

Restricciones Técnicas
No se aceptarán proyectos que:
tengan todo el código en un único archivo JavaScript
utilicen variables globales innecesarias
repitan lógica entre funciones
utilicen alert() como interfaz principal
recarguen la página para actualizar datos
usen librerías o frameworks no autorizados
Se recomienda una estructura similar a:

/
│── index.html
│
├── css/
│     styles.css
│
├── js/
│     app.js
│     astronautas.js
│     misiones.js
│     recursos.js
│     ui.js
│
├── assets/
│     img/
│     icons/
